Это протокол который шифрует данные между клиентом и сервером, сегодня на практике чаще используется  [[TLS]] шифрование, так как оно более современное и безопасное. Он нужен для того что бы создать доверенное подключение между клиентом и сервером и информация пользователя не могли получить злоумышленники.

Когда клиент видит специальный сертификат, он получает информацию о подлинности сайта, так же сайты которые получили ssl сертификат поднимаются выше в ранжировке поисковой системы и конечному клиенту проще доверять такому сайту. 

### Как получить ssl сертификат для сервера
1. Бесплатно (через Let's Encrypt)
	- Самый популярный вариант.
	- Сертификат выдаётся автоматически и обновляется каждые 90 дней.
	- Настраивается на сервере через инструменты вроде:
		- certbot (для Apache, Nginx)
		- Панели хостинга (например, cPanel, Plesk)

2. Платный сертификат
	- Покупается у сертификационного центра (Comodo, DigiCert, GeoTrust и т.д.).
	- Может содержать подтверждение организации (OV/EV).
	- Используется крупными компаниями, банками и т.д.

3. Через хостинг-провайдера
	- Многие хостинги автоматически выдают SSL при подключении домена.
	- Обычно делается в один клик: «Включить HTTPS» или «Добавить SSL-сертификат».

# Практическая инструкция получения ssl сертификатов

### Certbot (официальный вариант)
Скачивание cerbot с зависимостями:
```bash
apt-get update && apt-get upgrade && apt-get install certbot python3-certbot-nginx
```

Только выдача файлов:
```bash
sudo certbot certonly --standalone -d example.com -d www.example.com
```

после установки сертификатов они появляются в директории:
```
/etc/letsencrypt/live/example.com/
```
